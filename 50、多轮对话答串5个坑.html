5 个坑 · 点开看怎么填
每个坑点一下,看症结 + 怎么填 + 能直接抄的默认做法。
1
指代没消解
第二句"它能退吗",你直接拿"它"去检索,啥也搜不到 —— 60% 的追问都有这种没还原的代词。
填法 ▸
怎么填:检索前先做 query 改写,把"它、那个、这个"还原成上一轮的具体实体。默认:改写"它能退吗"→"这件冲锋衣能退吗"再检索
2
省略没补全
用户说"那海外呢?",省了主语和动作,直接拿去检索,召回一片空白。
填法 ▸
怎么填:改写时把省略的成分补回来,凑成一个独立完整的问题。默认:"那海外呢"→"海外的退款政策是什么"
3
历史一股脑全塞去检索
把整段对话拼起来扔进检索,噪声淹没当前意图,反而比单轮还不准。
填法 ▸
怎么填:别塞全历史。用历史去"改写"出一个独立完整的 query,只拿这条干净的 query 去检索。默认:历史 → 改写一句 → 用改写句检索
4
答串、会话串味
把上一个用户、或者上一个话题的上下文,串进了这一次的回答里 —— 最尴尬的翻车。
填法 ▸
怎么填:会话严格隔离,按 user + session 分开;再加话题切换检测,旧话题别往新问题里带。默认:按 user+session 隔离 + 话题切换检测
5
意图漂移没跟上
用户聊着聊着换了话题,系统还抱着前面那摊旧上下文,答得驴唇不对马嘴。
填法 ▸
怎么填:检测到意图/话题切换,就重置或缩减历史,别让旧上下文拖累新问题。默认:话题切换 → 重置/缩减历史窗口
单轮 vs 多轮 · 差在哪
同一套知识库,差别全在"检索之前怎么处理这句话"。
对比项单轮(准)多轮·没处理(崩)
检索输入一句完整问题"那它能退吗"带代词的半句
历史怎么用不需要要么不用、要么全塞(都错)
会话边界天然隔离容易串到别人/别的话题
解法直接检索检索前先 query 改写
怎么用 · 走一遍
同一段对话,加了 query 改写之后是怎么答对的。
第 1 轮:这件冲锋衣多少钱?　第 2 轮:那它能退吗?
1
改写:结合历史,把"它"还原成"这件冲锋衣" → "这件冲锋衣能退吗"。
2
用改写后的独立完整 query 去混合检索,命中"冲锋衣退货政策"。
3
生成时带引用,基于检索内容作答。
答:这件冲锋衣支持 7 天无理由退货,依据《退货政策》第 2 条。
(没接住上文?那就别猜,反问一句"您是指刚才的冲锋衣吗"。)
深一层 · 改写本身也会翻车
改写是解药,但改写错了会放大错误 —— 这 3 个二级坑也得防。
A
改写过度,把意思改歪了
用户问"那退货呢",改写器自作主张补成"那退货运费谁出",问题被换了。
怎么防 ▸
怎么防:只做"最小改写"——只还原代词、补省略,不许新增用户没说的限定。原则:minimal rewrite,只补不改
B
改写丢了关键信息
把多轮压成一句时,把上文的"仅限会员"这种关键限定漏掉了,检索就偏了。
怎么防 ▸
怎么防:改写时保留上文里的限定词、实体、数字;拿不准就把原句也一起送检索做并集。默认:改写句 + 原句 双路召回合并
C
改写置信低还硬上
指代有歧义("它"可能指两样东西),改写器瞎猜一个,直接错到底。
怎么防 ▸
怎么防:给改写打个置信度,低于阈值就回退用原句、或者直接反问用户"您是指 X 吗"。默认:置信低 → 回退原句 / 反问澄清
PM

拍板默认值(可抄)

直接抄去跟工程对
① query 改写 —— 检索前必加。指代消解 + 省略补全,用小模型改写
② 改写兜底 —— 改坏了别硬上。改写置信低 / 跑偏就回退用原句
③ 会话隔离 —— 杜绝串味。按 user + session 隔离上下文
④ 话题切换 —— 别抱旧上下文。检测意图漂移 → 重置/缩减历史
⑤ 改写进 Eval —— 它也会出错。多轮 case 进 golden set,盯改写准确率
