老板问数据 · CFO 问烧多少 · 客户问准不准
9 成 PM 慌的根子不是没数 · 是没体系
场景 01 · 周二一早
老板冲你竖大拇指:
「昨天数据怎么样?」
→ 你拉哪张图?
场景 02 · 十分钟后
CFO 端着咖啡:
「这 AI 一天烧多少钱?月度预算扛得住吗?」
→ 你五秒答得出来吗?
场景 03 · 半小时后
大客户来电话:
「我们试了上午有几个不太准 · 你们准确率多少?」
→ 你给的是承诺还是数据?
反认知 · 传统 KPI 不够看

传统 KPI 看不到 AI 产品真问题

9 成 PM 上线第一周都错在这
老板要你拉数据 · 你打开的是 DAU + GMV
  • DAU 涨 ≠ AI 在创造价值(可能是营销带来的)
  • GMV 涨 ≠ AI 比规则强(可能是季节性)
  • 留存好 ≠ 用户满意(可能被押金锁住)
  • 点 👍 多 ≠ AI 真有用(用户懒得点 👎)
AI 产品有自己的健康指标 · 不是传统 KPI 能盖住的
→ 7 张 dashboard 缺一不可
7 Dashboards · 一张一张拆

7 张 dashboard
看什么 · PM 视角 · 阈值 · 真实案例

每张给你 4 件事 — 看什么指标、PM 视角(怎么解读)、告警阈值、真实案例(谁靠它救命 / 谁漏了它翻车)。

01
用户激活 + 留存 dashboard
基础款 老板必问
看什么
DAU + 新用户激活率 + Onboarding 完成率 + Day 1 / Day 7 留存
PM 视角
DAU 涨 = 老板开心 · 但 Day 1 留存才是命。Onboarding 完成率低 = 用户进来没用上 AI 就走 — DAU 里算数 · 但他没拿到价值
告警阈值
Day 1 留存 < 40% 立刻拉数据 · Onboarding 完成率 < 70% 立刻拉 funnel 找流失
02
AI 任务完成率 dashboard
AI 专属 传统产品没有
看什么
任务完成率(拿了答案且没换问法没找人工)+ 平均轮次 + Power user 占比
PM 视角
AI 答了 ≠ 用户满意 · 任务完成才算用户拿到价值
告警阈值
完成率 < 60% = AI 在做表面文章 · 平均轮次 > 2.5 = 用户反复改问法体验差
真实案例 某企业知识库 DAU 涨得不错老板高兴 · 但任务完成率只有 42% · 10 个用户 6 个用完啥也没拿到 · 问题不在 DAU 在「为啥用了不解决问题」
03
AI 满意度 dashboard
客户必问
看什么
👍 / 👎 比例 + 负面关键词(「不准」「胡说」「不懂」)+ NPS
PM 视角
用户嘴上不说 · 但他用「行为」告诉你 — 点 👎 + 写评论
告警阈值
👎 > 5% 立即排查 · > 10% 进 emergency review
真实案例 某客服👎 一直 3% · 某天突然涨到 12% · 一查发现前一天工程师悄悄换了模型版本 · 新版本对方言语义差 · 👎 dashboard 救了他们 — 否则用户大规模流失才发现
04
兜底 + 求助率 dashboard
最容易漏 隐性差评
看什么
30 秒内换问法占比 + 转人工点击率 + 人工工单类型(反推 AI 哪里没答上)
PM 视角
用户没点 👎 · 但用户「行为」告诉你 AI 没解决 — 这是隐性差评
告警阈值
换问法 > 30% = AI 在装会答 · 转人工点击率涨 = 用户对 AI 失去信任
真实案例 某 SaaS 助手👍 看着 90% 老板满意 · 但换问法占 42% — 表面 NPS 好看 · 底子已经裂了 · 没点 👎 不等于没差评 · 用户懒得点 · 但他用脚投票
05
成本 dashboard
CFO 命门
看什么
Token 月消耗 + 单用户日均成本 + 月度预算燃尽率 + 单次调用成本(模型 / 检索 / embedding / 缓存 / 兜底分层)
PM 视角
CFO 一定问「一天烧多少」「这个月扛得住吗」「明年要多少预算」· 你能不能5 秒答出直接决定你这 PM 的专业度
告警阈值
月度燃尽 > 60%(月中刚过)立即告警 · 单用户日均超阈值 = 查是不是有人在刷接口
真实案例 某 AI 客服预算月初定 50w · 月中 15 号燃尽率 72% · PM 没盯 dashboard · 月底超支 82w 被 CFO 拉去复盘 · 月中那天看见破 60% 立刻启动「降级 prompt + 缓存优化」完全来得及
06
响应延迟 dashboard
C 端体验
看什么
P50(一半用户体验)+ P95(95% 用户体验)+ P99(最慢 1%)+ 超时率
PM 视角
用户对 AI 延迟容忍 2 秒上限 · 超 2s 开始烦 · 超 5s 直接走
告警阈值
P95 > 2s 立即告警 · 超时率 > 0.5% 立即告警
真实案例 某 C 端聊天产品 DAU 突然下跌 PM 找不到原因 · 一查 P95 从 1s 涨到 3.8s · 后端为省成本切了便宜模型 · 用户用脚投票 · 延迟 dashboard 早盯能早一周发现
07
异常 + 兜底链 dashboard
出大事前兆
看什么
报错率 + 兜底链触发率 + 上游 API 错误率(OpenAI / Claude / Gemini 各家)+ 限流次数
PM 视角
每项单独看不严重 · 但4 项同时动 = 产品快炸了
告警阈值
报错率 > 0.1% 立即告警 · 上游 API 错误率 > 1% 自动切多模型
真实案例 2026-06-12 Anthropic 全球下架 Fable 5 + Mythos 5 · 如果你那天用 Fable 5 · 产品瞬间瘫痪 · 怎么知道「自动切到 Opus 4.8 备用」?就看这张 dashboard · 上游 API 错误率突破阈值 → 自动触发模型切换 → 用户无感
