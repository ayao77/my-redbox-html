Computer Use
看似神奇商用化有 3 个死穴

By NOVA · PM 视角讲透 · 不是技术演示
90% 演示都是骗你的
Demo 视频里 AI 顺利操作浏览器——
你以为能直接落地公司？
实测准确率 95% = 生产灾难
每 20 次操作 1 次错。这放在 demo 里看不出来，放在生产每天 100 次操作 = 每天 5 次事故。AI PM 要的是"在哪能真用、怎么兜底"。
01
PART · ONE
PM 视角的场景判断

Computer Use 不是"哪里都能用"的能力。先看你的场景在哪边——

SUITABLE
✓ 该上 Computer Use
无 API 的老系统——只有图形界面的 ERP / 财务软件
低频高价值——一次操作省 30 分钟，每天跑几十次
允许有人监督——AI 操作时人在旁边能随时叫停
错了能撤回——比如填表草稿、整理数据，不是真支付
跨多个工具——Excel + 邮件 + 网页混着用，写脚本太麻烦
DON'T USE
✗ 别碰 Computer Use
高频高并发——每秒处理多个请求，成本会爆炸
有现成 API——能直接调 API 就别截图操作，慢 50 倍
关键金融操作——转账 / 撤销订单 / 删数据，错了要赔钱
对延迟敏感——客服实时回复、推荐系统，AI 操作太慢
责任归属模糊——出事说不清是 AI 错还是用户错

Anthropic / OpenAI 的 Computer Use demo 都是精心挑选的最佳案例，让人看完想立刻上。

但真生产环境——平均 1 次操作 15 秒 + 调用大模型 + 每步截图理解，成本是普通 API 的 50 倍。每天跑 1000 次操作，月烧 5-10 万。

PM 第一件事——算清楚 ROI，再决定是否启动。

02
PART · TWO
3 种真能落地的产品形态

光会"AI 操作电脑"没用——必须包装成用户敢用、买单的产品。3 种真实形态。

FORM 01 人机共驾Co-pilot Mode 最稳
本质 AI 提议操作步骤，每一步等用户确认才执行。用户全程能看见、能改、能停。
类比 特斯拉的 L2 自动驾驶——AI 在开，但人手不能离方向盘。
案例 Anthropic 的 Computer Use Demo、Adept 的 Workflow Pilot——AI 边操作边解释，用户随时叫停。
PM 收益 责任清晰——出错是用户没及时叫停。商用最稳的形态。
FORM 02 沙箱代办Sandbox Agent 最快试
本质 AI 在隔离虚拟机里跑——不碰真实业务系统。结果用户审核确认后再迁移。
类比 让实习生在测试环境试错，确认无误再写进生产。
案例 Cognition 的 Devin（在沙箱编程）、Browserbase（AI 在云端浏览器里操作）——错了不影响真生产。
PM 收益 用户敢让 AI 跑——有事不影响真实业务。这是 ToB 落地最受欢迎的形态。
FORM 03 任务分级路由Hybrid Routing 最省钱
本质 大部分任务走 API + Function Calling（快 + 便宜）。只有需要操作 GUI 时才上 Computer Use。
类比 能走高速就不走小路。Computer Use 是"备用方案"，不是主路。
案例 飞书 / Notion AI 大量用 API，只在第三方老系统才上 Computer Use 接管。
PM 收益 综合成本砍 80%——把贵的能力用在刀刃上。
PM 心法

共驾 → 安全敏感场景（金融 / 法务 / HR）

沙箱 → 探索性任务（数据分析 / 内容生成）

路由 → 大规模生产（客服 / 自动化办公）

真高级 PM——三种混合用。不同任务用不同形态。

03
PART · THREE
3 个真实翻车 + 兜底设计

下面 3 个翻车场景——每个都是 PM 必须提前设计兜底机制的。否则上线第一周就会被业务方喊停。

CRASH 01 · 误操作
AI 把客户支付订单按错，触发误退款
SCENE · 财务自动化场景
为啥发生
AI 截图理解时把相邻两个按钮位置认错——本来该点"确认"，点成了"退款"。这种错误在 demo 视频里看不出，但生产每 20 次 1 次发生。
兜底设计
高风险操作必须二次确认机制——AI 点完"退款"前，弹一个对话框让用户确认。或者更彻底——白名单 机制，AI 只能点 PM 预先批准的按钮，"退款"按钮永远不在白名单里。
CRASH 02 · 太慢
客户等 3 分钟看 AI 慢吞吞操作
SCENE · 客户支持场景
为啥发生
Computer Use 每步要截图 + 大模型理解 + 决策。一个简单的"查订单 + 改地址"流程要 20+ 步，真跑下来 2-3 分钟。客户在客服窗口等不住，刷屏催。
兜底设计
① 实时进度条——告诉用户"AI 正在第 5/12 步"，等待心理预期降下来。
② 异步队列——常规任务进队列，几分钟后通知"已完成"，不让用户傻等。
③ 高频任务直接退回 API 模式，Computer Use 只做兜底。
CRASH 03 · 太贵
CFO 看到月账单——15 万API 费
SCENE · 内部办公自动化
为啥发生
公司 200 人都开始用 Computer Use 自动化日常任务——每人每天 50 次操作 × 200 人 = 1 万次/天。Computer Use 单价是普通 API 的 50 倍，月成本爆炸。
兜底设计
① 预算硬墙——按部门 / 用户配额度，超了自动停。
② 缓存高频操作——相同任务的操作脚本录制一次复用，下次不用 AI 重新理解屏幕。
③ 定期 Audit——每月看哪些任务 ROI 低，砍掉或退回脚本化。
面试金句 · 直接抄
被问"Computer Use 怎么落地"——
先算账——单次成本是普通 API 的 50 倍，先算 ROI
先选形态——共驾 / 沙箱 / 路由，按业务风险选
先设兜底——二次确认 + 白名单 + 预算硬墙
先小范围——选 1 个低风险场景 POC，跑通再扩

4 句答完，面试官知道你不是被 demo 带节奏的小白

核心结论 · TAKEAWAY
Computer Use 不是万能钥匙——
是"兜底能力"
