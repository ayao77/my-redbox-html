AI PRD 不是
传统 PRD ＋ 一段提示词

真正的差别 · 在这 3 块

By Nova

很多人以为，AI PRD 就是 传统 PRD 加一段 prompt 设计。这个理解，差得很远。我带学员一年多，看过两百多份 PRD——AI PRD 和传统 PRD 真正的差别，不在表面，而在三个被反复忽略的结构性差异上。

真正的差别 · 有 3 个
壹
CHAPTER 01 / 03 · DIFF 01
多了「Agent 核心能力设计」这一整块

AI PRD 的第一块结构性差异，是必须包含一整块 Agent 核心能力设计——不是一句"接入大模型"带过，而是要拆成 6 个明确的子模块，每一个都要落到具体设计。

agent.core_capabilities 6 modules
01 提示词 Prompt
02 工作流 Workflow
03 知识库 Knowledge Base
04 任务链 Task Chain
05 对话状态 Dialogue State
06 上下文预处理 Context Pre-proc
贰
CHAPTER 02 / 03 · DIFF 02
多了「多 Agent 协同」架构引擎设计

第二块差异，是大厂的 AI 产品基本都不是单 Agent，而是多 Agent 协同。PRD 里必须画清楚三件事：

01 主 Agent ↔ 子 Agent 怎么分工
02 消息怎么传递
03 冲突怎么解决
CRITICAL · 关键判断
面试时最容易被深挖的部分
↓ 答不上来 = 二面挂
叁
CHAPTER 03 / 03 · DIFF 03
多了「评估体系」与「大模型选型」

第三块差异，是评估体系和大模型选型——这两块在传统 PRD 里完全没有对应物。

traditional_prd.equivalent → null
EVALUATION_SYSTEM · 评估体系
▸ 01 任务级 task_level
▸ 02 对话级 dialog_level
▸ 03 业务级 business_level
MODEL_SELECTION · 模型选型
▸ 01 为什么是这个模型 why
▸ 02 benchmark 怎么跑 how
▸ 03 成本怎么算 cost
