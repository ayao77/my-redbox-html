PART · ONE
完整数据流架构
架构图 · v3 稳定版
蓝 · INPUT3 路输入数据源
黄 · CORE向量检索 / 智能体核心
绿 · OUTPUT最终输出

完整数据流 · 6 步走完一次请求
STEP
01

语义检索召回

用户需求"我想选爆款冬装" → Embedding 向量化 → 在向量库召回 Top 100 相关商品。

STEP
02

3 Agent 并行调用

100 个候选商品同时进 3 个 Agent。数据 Agent 算销量利润 / 趋势 Agent 算流量势能 / 库存 Agent 算供应链可行性。

STEP
03

各 Agent 输出评分

每个 Agent 给出自己视角的评分 + 推理理由，统一用 JSON 结构化输出，方便聚合。

STEP
04

Supervisor 聚合仲裁

主控按权重 数据 40% / 趋势 35% / 库存 25% 加权。3 个意见冲突时优先库存（保供应链）。

STEP
05

Top 20 输出

按综合分排序，输出最终 Top 20 选品清单——含商品 ID、推荐理由、预期销量、风险点。

STEP
06

反馈飞轮回路

真实购买数据 → 反哺 3 个 Agent 的权重。每周自动调权——这就是越用越准的秘密。

02
PART · TWO
3 个 Agent 各自做啥
AGENT 01 数据 AgentData Analyst
GPT-4o Function Calling
ROLE 销量、转化率、利润率、退款率 的定量分析师。给每个候选商品打"数据维度分"。
TOOLS query_sales 近 30 天销量 / get_margin 利润率 / refund_rate 退款率
OUTPUT { score: 0-100, reasons, risks }
WEIGHT 40%占比最大——"钱"最实在
AGENT 02 趋势 AgentTrend Watcher
Claude 3.5 Web Search
ROLE 季节、热搜、竞品 的嗅觉型判断者。给候选商品打"势能分"——能不能成爆款。
TOOLS get_hot_keywords 平台热搜 / competitor_scan 竞品 / seasonal_curve 季节系数
OUTPUT { score, trend: "rising/stable/declining", reasons }
WEIGHT 35%捕捉"未来值"——提前 3 个月预判
AGENT 03 库存 AgentSupply Auditor
GPT-4o-mini SQL Tool
ROLE 备货、供应、物流可行性 的风控守门员。一票否决高风险商品。
TOOLS check_stock 现货 / lead_time 补货周期 / logistic 仓配能力
OUTPUT { score, blocker: true/false, reasons }
WEIGHT 25%+ 一票否决权——库存出事项目就挂
为啥不用1 个超大 prompt？
因为 "专业分工 + 单点可优化"

如果只用 1 个 Agent——发现"趋势判断不准"，你不知道改哪句 prompt。

但用 3 个 Agent——趋势出问题，只优化 Agent 02 的 prompt + tools，其他不动。

单点优化 + 单点回归——这就是 AI 产品工程化的核心。

03
PART · THREE
从 38% → 67% 的 3 次迭代

这套架构不是一上来就这样。从 v1 单 Agent 翻车，到 v3 多 Agent 上线，每次迭代都源自一次"事故 + 复盘"。

v1.0
单 Agent 暴力解法 2024 Q1
架构：商品 + 用户行为 → 1 个超大 prompt → 选品列表
准确率38%
GMV-5%
运营接受度12%
FAILURE · 翻车点
一个 prompt 同时考虑数据 + 趋势 + 库存，模型注意力分散——经常"漏掉库存信息"导致选了缺货爆款。
v2.0
引入 Embedding + 拆 2 Agent 2024 Q2
架构：商品库 → Embedding 向量检索 → 2 Agent（数据 + 趋势）→ 简单加权
准确率54%
GMV+6%
运营接受度45%
NEW ISSUE · 新问题
数据 + 趋势都对了，但库存这一层没人管——上线 1 个月推荐了 30+ 缺货商品，仓储部门来吵。
v3.0
3 Agent + Supervisor + 数据飞轮 2024 Q4
架构：3 路输入 → Embedding → 3 Agent 并行（数据/趋势/库存）→ Supervisor 加权聚合 → 反馈飞轮
准确率67%
GMV+12%
运营接受度78%
延迟2.4s
面试金句 · 直接抄
被问"为啥用多 Agent 不用单 Agent"——
专业分工——每 Agent 角色单一，prompt 更精准
单点可优化——某 Agent 出问题只改它，迭代快 5 倍
并行计算——3 Agent 同时跑，延迟降 60%
风控隔离——库存 Agent 一票否决，避免单点失误带崩
