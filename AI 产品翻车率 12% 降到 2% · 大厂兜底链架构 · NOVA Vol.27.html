组件 01

Confidence 评分 · 模型怎么知道自己不确定

模型给一个回答,先评估自己有多确定。确定就直接给用户,不确定就走兜底链。3 种工业级方法 — 点击切换看每种的精度 / 成本 / PM 决策。

精度 ★★ / 5 成本 0 延迟 0

OpenAI / Claude API 返回每个 Token 的概率(logprobs 参数)。把 Token 概率算综合分 = 这次回答的置信度。最常见的工业基线方法。

致命缺陷:Logprobs 反映"语言流畅度",不反映"事实正确性"。模型胡说八道也可以说得很流畅 — Confidence 0.95 但答案完全错的事故根源。

通用场景(电商、HR、内容生成)首选。重要场景必须叠 Self-Consistency 或 LLM Judge 校验。
精度 ★★★★ / 5 成本 3x 延迟 3x

同一个问题让模型跑 3 次。3 次回答一致 → 高置信。不一致 → 低置信。本质是多次采样投票。

实现:temperature=0.7 跑 3 次,用规则或语义相似度判定一致性。Token 成本 3 倍但准确率显著提升 10-20pt。

高风险场景(医疗 / 金融 / 法律)的标配。重要决策类回复必上。
精度 ★★★★ / 5 成本 中 延迟 2x

第 2 个模型给第 1 个模型打分。Judge Prompt 写好维度(准确性 / 安全性 / 相关性),输出 0-10 分。

最大优势:可以做多维度细分(不只是"对错",还有"语气友好""有没有越界")。

超高风险 + 主观维度场景。配 Self-Consistency 做组合拳。

组件 03

三层兜底链 · 80% / 15% / 5%

这一段是本期最干货。所有翻车场景里,80% 都是这块没做对。兜底链分层 = 成本分层。

L1
80%
自动降级用户无感· Automatic Fallback
  • 切换更强模型 — GPT-3.5 → GPT-4
  • Self-Consistency 重试 3 次 取多数
  • Few-shot 增强 — 临时注入相似 case 再试
💰 成本 · 模型升级 + 重试 · 比转人工便宜 10 倍
L2
15%
规则兜底用户感知低· Rule Fallback
  • 预设话术 — "请提供更多信息" / "稍等查询"
  • FAQ 推荐 — "您是不是想问 X?"
  • 推荐路径 — 引导到知识库 / 帮助页
💰 成本 · 几乎 0 · 都是预设内容
L3
5%
人工兜底用户感知高· Human Fallback
  • 转人工客服 — 同步
  • 异步处理 — 1 小时内回复
  • 高级 PM 介入(P1 场景)
💰 成本 · 高(人力)· 必须留给真正必要的场景

心法 — 80/15/5 比例错了 → 体验崩 或 成本失控。L1 优先,L1 处理不了走 L2,L2 不行才走 L3。

PITFALL

3 种典型翻车与 PM 兜底

01模型过度自信(最致命)
Confidence 0.95 但答案完全错。Logprobs 反映"语言流畅度",不反映"事实正确性" — 模型胡说八道也可以说得很流畅。大厂 P0 事故 80% 都是这一种。
重要场景叠 Self-Consistency + LLM Judge。不要单靠 Logprobs。Self-Consistency 跑 3 次投票,投票一致才算高置信。
02兜底链不分层
所有兜底都直接转人工 → 月成本 30 万。没有 L1 L2 设计,直接跳 L3。
80% / 15% / 5% 比例硬规则。L1 优先,L1 不行走 L2,L2 不行才 L3。每月做 L1/L2/L3 比例分析,失衡立即调整。
03触发率监控缺失
上线 1 个月后触发率从 8% 偷偷涨到 25%。原因 — 用户分布漂移,但没人盯。
触发率 + 触发原因分布做日报。涨幅 > 5pt 自动告警。每月 Eval 集回归测试,触发率失控倒查 Prompt / 检索质量。
