8 成团队选错

2026 上 AI 客服别再问做不做了
是 选 RAG 还是 Agent

我做过 3 个 AI 客服项目。一个月烧 2.8 万,一个月烧 28 万,一个月烧 6.5 万。差的不是流量,是架构选型。Agent 不是 RAG 的升级版,是完全不同的产品形态 — 贵 10 倍。

今天用 PM 视角讲透 3 件事
  1. 三种架构的本质差异 — 它们不是代差,是形态
  2. 选型三维度 — 复杂度 / 流量 / 客单价
  3. 每种架构的 PM 必踩翻车
01
三种架构的本质差异 · 不是代差,是形态

很多人以为 Agent 就是 "RAG 加上 Tool"。错。本质完全不一样。三种各有适合的业务画像,不是 "代差升级" 关系。

RAG 客服

自助超市导购

智能搜索 + 改写答案 · 单轮无规划
能告诉你东西在哪
不会替你付钱
问答型场景
2-5 万/月中等流量 · 月成本
AGENT 客服

私人助理

搜索 + 决策 + 调工具 + 多步串联
能下单付款退货
全程都办了
多步操作场景
15-50 万/月同流量 · 贵 10 倍
混合架构

医院分诊

意图分流路由 · 双路径并行
小病自助挂号
大病转专家
2026 主流方案
5-10 万/月省 70% + 解决率 +30%

心法 — 强行用 Agent 解决退货政策这种单步问答 = 直升机送外卖,能送到但烧死你。

02
选型三维度 · 不靠手感,靠 3 个数字
3 个数字决定路线
A复杂度
RAG 够单步问答 · 退货政策 / 物流 / 发票
必上 Agent多步操作 · 查日志 / 改配置 / 串联系统
B流量量级
RAG 主路径高频 > 1000 单/日 · Agent 月烧扛不住
Agent 全上低频高价 < 100 单/日 · 客单价 > 1 万
C客单价
商业不通客单价 < 100 元 · 纯 Agent 别想
放心上客单价 > 5000 元 · Agent 成本占比 < 5%

如果业务又高频又复杂?混合架构 — 80% 流量走 RAG,20% 升级 Agent。这是 2026 的主流方案,我对接的跨境项目就这么做,月省 70% 成本同时拉高 30 分解决率。

03
三个真实案例 · 三种画像分别选了什么
CASE A 中型电商 · 选 RAG 高频低复杂 / 中等流量
业务日均 800 单咨询 — 退换货 / 物流 / 发票为主
方案RAG + 工业级三招(Rerank + Hybrid + 父子分块)
数据准确率 87% · 转人工率 18% · 月成本 2.8 万
教训高频低复杂场景,RAG 足够,Agent 是过度设计
CASE B SaaS 厂商 · 选 Agent 多步操作 / 高客单价
业务技术工单 — 查日志 → 排查 → 重启 → 通知 一条龙
方案Agent + Tool(查日志/监控/通知 API)+ MCP 标准化
数据首次解决率 78% · 月成本 28 万 · 仅给付费客户
教训多步真实操作只能上 Agent,但商业模型必须 cover 成本
CASE C 跨境出海 · 混合架构 2026 主流方案
业务80% 售前简单问答 + 20% 售后复杂处理
方案意图分流 — 简单 RAG · 复杂升级 Agent · 统一回答模板
数据准确率 85% · 月成本 6.5 万 · 比纯 Agent 省 70%
教训纯架构是过渡形态,混合架构才是 2026 主流
04
三种架构的 PM 防翻车

无论选哪个,都有特定的必踩翻车。PM 上线前必须写进 PRD,不是写进事故复盘报告。

RAG
翻车
RAG 客服三个死穴
多步问题模型硬编 · 知识更新滞后 · 多轮上下文丢失
多步问题强制转人工或升级 Agent · 文档变更自动触发索引重建 · 多轮场景用对话摘要压缩
Agent
翻车
Agent 客服三个死穴
Tool 调用错 · 循环死锁 · 成本爆炸(月烧超预算 3 倍)
每个 Tool 配高质量描述+示例 · 设最大迭代轮数(默认 5 轮) · Token 仪表盘+预算告警 · 超预算自动降级 RAG
混合
翻车
混合架构三个死穴
意图分类错路 · 双路体验不一致 · 运营复杂度翻倍
意图分类器单独建 Eval(准确率 90%+) · 两条路径强制统一回答模板 · 开两套监控仪表盘 + 错误归因机制
INTERVIEW GOLD · 高频题
你的 AI 客服选什么架构?
第 1
看业务形态 — 高频低复杂走 RAG · 多步真实操作走 Agent · SaaS 长尾走混合。
第 2
用三维度判断 — 复杂度、流量、客单价。不靠手感靠数字。
第 3
做商业账测算 — RAG 月 2-5 万,Agent 月烧 15-50 万,差 10 倍。算不过来不上 Agent。
第 4
每种架构提前写翻车兜底 — RAG 防硬编、Agent 防死锁、混合防分类错路。
4 句答完,面试官知道你不是被 Agent 概念带节奏的小白,是算过账上过线的 PM。
ONE-LINE
Agent 是产品上限 · RAG 是商业底线 · 混合是 2026 主流
