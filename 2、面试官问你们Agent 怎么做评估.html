面试官问
"你们 Agent 怎么
做评估？"

我学员是这么答的
By NOVA · 真实带教案例 · 字节面试通过
核心判断
AI 产品评估的差别
不在指标多不多，在颗粒度
传统盯转化、留存、ARPU。
AI 产品必须拆到 3 层——只答指标名字 = 普通候选人。
LAYER 01
壹
FOUNDATION · 第 01 层
任务级评估

最基础的一层——单次任务能不能完成。比如客服 Agent，看任务完成率、首次解决率、转人工率。很多人答到这就停了，但这只是评估的入口。

任务完成率 completion_rate
单次任务是否被 Agent 顺利完成
核心
首次解决率 first_resolve
用户第一次就拿到满意结果的比例
核心
转人工率 handoff_rate
Agent 能力边界的真实信号
信号
LAYER 02
贰
MIDDLE · 第 02 层
对话级评估

多轮对话里 Agent 表现怎么样——上下文有没有丢、有没有幻觉、回答有没有相关性。这一层必须做人工标注 + 自动化打分双路并行。

上下文保持 context_retention
多轮中信息没有丢失或被覆盖
核心
幻觉率 hallucination
回答是否包含编造、错误信息
风险
相关性打分 relevance
回答是否切题、是否回应用户真实意图
质量
加分点 · BONUS
学员特意提了 LLM-as-a-Judge 这个方法
面试官当场点头！
LAYER 03
叁
TOP · 第 03 层
业务级评估

Agent 上线后，对核心业务指标有没有真实影响。这一层最容易被忽略，但恰恰是判断你有没有产品 sense 的关键。

整体满意度 CSAT
上线前后用户满意度变化
业务
平均处理时长 AHT
单次会话耗时是否下降
业务
人力成本 cost_saved
节省的客服坐席与 ROI
ROI
真实面试现场 · CASE
— 面试官追问 — "评估体系是一次性设计完的吗？"
→ 学员答 "不是。要随着产品迭代不断调指标"
接着他答了三个迭代节点的真实例子。
那一面 → 直接过了！🎉
核心结论 · TAKEAWAY
别只答指标名字
要答 为什么、怎么测、怎么调
这才是 AI PM 的颗粒度
也是面试官判断"真懂还是假懂"的分水岭
