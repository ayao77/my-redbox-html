026 种死法:语音 Agent 在哪翻车
这期核心。每种配 场景 → 根因 → 兜底。
被打断不停
场景用户"不是不是,我问另一个",AI 没停,俩人一起说,用户更烦。
根因没做 barge-in,没实时监听用户声音。
兜底支持 barge-in(检测到用户说话立刻停 TTS)+ 调好 endpointing 阈值 + 全双工监听。
量化盯 打断处理成功率:用户插话后多少毫秒内停下来。
延迟 / 尴尬沉默
场景用户问完,AI 那头 3 秒没声,"喂?喂?"以为断了,挂了。
根因ASR→LLM→TTS 串行、不流式,链路太长。
兜底三段全流式 + 填充语("让我帮您查一下~"先出声)+ 常见问题缓存 + 简单意图走小模型。
量化盯 端到端延迟 P95(看最慢那批,压到 1 秒内)+ 沉默占比。
听错 / 口音 / 数字
场景用户报订单号"1387",AI 听成"1380";方言、地铁噪音、专名识别错,下游全错。
根因ASR 在噪音、口音、专名、数字上词错率飙升。
兜底领域热词增强(产品名/订单号)+ 关键数字复述确认("订单 1387 对吗")+ 听不清降级转文字/短信/人工。
量化盯 ASR 词错率 WER,单独看数字/订单号字段,别用整体平均盖住。
该转人工不转(情绪)
场景用户已经骂人了,AI 还在"很抱歉给您带来不便"机械应答,火上浇油。
根因没有情绪识别 + 升级触发,AI 不知道该退场。
兜底情绪/关键词识别("投诉""转人工")→ 立刻转人工 + 无缝交接带上下文。
量化盯 转人工率 · 升级及时率:该转的有没有及时转。
乱承诺 / 幻觉(语音版红线)
场景AI 电话里张口"给您退款""三天到账""保证解决",政策根本不是这样。
根因政策/价格/时效靠模型编,语音又难审计、收不回。
兜底政策/价格/时效只读知识库不编 + 关键承诺转人工或二次确认 + 全程转写+录音审计。
量化盯 红线触发率 · 漏放率:违规承诺有没有被放过。
合规 / 录音 / AI 披露
场景没告知就录音、没说明是 AI、声纹隐私没处理,踩消保和隐私红线。
根因把线上文字客服那套直接搬到电话,漏了语音特有合规。
兜底开场告知"我是 AI 助手" + 录音告知 + 隐私/声纹合规 + 高风险双录/转人工。
量化盯 AI 披露率 · 录音告知率:会被监管查的硬指标。
03怎么搭:落地三步(顺序别错)
语音第一性是"及时",先压延迟再谈答得好。

先设红线和兜底

被问"语音 Agent 怎么落地",按 4 句答:
1先压延迟——三段全流式,端到端 < 1 秒,填充语兜沉默。
2先做轮次——barge-in 能被打断 + endpointing 不抢话不冷场。
3先治听错——热词增强 + 关键数字复述确认 + 听不清降级。
4先设红线——政策只读库、情绪转人工、AI 披露 + 录音合规。
4 句答完,面试官知道你不是被 demo 带节奏的小白。
